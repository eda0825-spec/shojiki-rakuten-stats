# SHOJIKI 客户反馈看板 - 工厂使用指南

> 日本 SHOJIKI 品牌为中国工厂提供的，整理楽天/Amazon 客户评论的看板。
> 网址: <https://eda0825-spec.github.io/shojiki-rakuten-stats/>

## 这是什么

楽天/亚马逊上日本顾客对您工厂生产的产品 (SH-J001 或 SH-J002) 写的评论，
经过 AI 自动整理后展示。每条评论都有:

- **日本語要約 / 中文摘要** — 一句话说明顾客在说什么 (双语)
- **分类** — 故障 / 改善需求 / 好评 / 提问 / 其他
- **严重度** — 高 / 中 / 低 (仅故障和改善需求有)
- **话题标签** — 涉及哪些零件或功能 (电池、滚轮、吸力等)
- **对策建议** — AI 提议的工厂端可以采取的措施

## 如何使用

### 1. 切换型号 (上方Tab)
- `SH-J001` - 吸力重视型
- `SH-J002` - 轻量&操作性型

### 2. 优先看这两个分类

**🔴 故障 (defect)** 红色徽章
顾客明确说"坏了/不能充电/有异响"。**这是工厂需要立即重视的**。
按严重度: 高→中→低 排列。

**🟠 改善需求 (improvement)** 橙色徽章
能用，但顾客觉得"太重/太吵/吸力再强点就好"等。
反复出现的同一话题 = 下一代改进的重点。

### 3. 用筛选定位问题
- **分类** 行: 点"故障"只看故障
- **严重度** 行: 点"高"只看严重问题
- **话题** (页面中部): 点"电池"只看电池相关
- **搜索框**: 输入关键词 (中日双语都可以)

### 4. 点击评论展开全文
评论文本框默认折叠 (140px)。点一下即可展开看完整原文。

## 看到值得报告给日本的问题时

1. 复制 中文摘要 + 中文话题
2. 通过现有沟通渠道 (邮件/微信/钉钉) 报告给日本工程师
3. 或者，等 Lark Base 配置完成后直接在 Base 上记录 (日本会同步设置)

## 数据多久更新一次

每天 北京时间 8:00 (= 日本 9:00) 自动从楽天抓取最新评论。
Amazon 评论由日本端定期手动上传 CSV。

## 故障 / 严重度 的判断标准

| 分类 | 严重度 | 例 |
|---|---|---|
| 故障 | 高 | 充电起火、漏电、零件脱落致伤 |
| 故障 | 中 | 电池不充电、电机不转、明显异响 |
| 故障 | 低 | 偶发卡顿、外观瑕疵 |
| 改善需求 | 高 | 多数用户觉得"很重"等共通不满 |
| 改善需求 | 中 | "希望吸力更强"等优化建议 |
| 改善需求 | 低 | 个人喜好范围 |

## 关于翻译的准确性

中文摘要是 AI (Claude) 自动翻译的。技术术语可能不完全准确，
有疑问时请对照日文原文，或向日本工程师确认。

## 看板访问不畅时

如果在中国境内 GitHub Pages 访问慢:
- 用 VPN/代理
- 或者要求日本端导出 JSON 文件通过其他渠道发送
